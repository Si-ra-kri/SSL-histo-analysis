# DCGAN on BreaKHis: Histopathology Image Generation

## Overview

This project implements a **Deep Convolutional Generative Adversarial Network (DCGAN)** using **PyTorch** to generate synthetic histopathology images from the **BreaKHis breast cancer dataset**.

Histopathology images are essential for medical diagnosis, but collecting and labeling large datasets is expensive and time-consuming. Generative models such as GANs can learn the distribution of real images and generate realistic synthetic samples that can be used for:

- Data augmentation
- Medical image research
- Improving training datasets
- Representation learning

This notebook explores how a DCGAN can learn the visual characteristics of breast cancer histology images.

---

## Dataset

The model is trained using the **BreaKHis (Breast Cancer Histopathological Image Classification) dataset**.

Dataset features:

- Breast tumor histopathology images
- RGB microscopic tissue images
- Multiple magnification factors
- Benign and malignant tumor samples

Dataset reference:

Spanhol et al., *A Dataset for Breast Cancer Histopathological Image Classification*, IEEE Transactions on Biomedical Engineering.

Dataset structure used in the project:

```
BreaKHis_v1/
└── histology_slides/
    └── breast/
```

## 3. DCGAN Architecture

The project implements the **Deep Convolutional GAN (DCGAN)** architecture.

### Generator

The generator converts a **random noise vector** into a synthetic histopathology image.

Key components:

- ConvTranspose2D layers
- Batch Normalization
- ReLU activations
- Tanh output layer

Input:

```
Random noise vector (latent space)
```

Output:

```
Synthetic histopathology image
```

---

### Discriminator

The discriminator classifies images as **real or generated**.

Key components:

- Convolution layers
- Batch Normalization
- LeakyReLU activations
- Sigmoid output

The discriminator learns to distinguish between:

- Real images from the dataset
- Fake images generated by the generator

---

## Training Process

GANs are trained through an **adversarial process**:

1. Train the discriminator on real images
2. Train the discriminator on generated images
3. Train the generator to fool the discriminator

Loss Function:

```
Binary Cross Entropy Loss (BCELoss)
```

Optimizer:

```
Adam
```

Typical parameters used in the notebook:

```
Epochs: 60
Latent vector size: 100
```

Training runs on **GPU if CUDA is available**, otherwise CPU.

---

## Generated Images

After training, the generator produces **synthetic histopathology images** from random noise.

Generated samples are visualized using:

- `torchvision.utils.make_grid`
- `matplotlib`

These generated images capture some of the texture and color patterns seen in histopathology slides.
